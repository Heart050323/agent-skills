# 人間らしい文章でレポートを書く

人が書いたと読めるレポートを仕上げる。下書きを作って終わりにせず、必ずリライト工程まで通す。

## 手順

### 1. 要件を固める

引数からテーマ・資料を取得する(ファイルパスなら読み込む)。以下が判別できない場合のみユーザーに確認する。

- 読者と提出先(大学の課題 / 業務報告 / 顧客向け など)
- 文体(学術=である調 / ビジネス=です・ます調)
- 分量と構成の指定

完了条件: 読者・文体・分量の3点が決まっている。

### 2. 構成を組む

レポートの主張(結論)を1文で定めてから、見出しレベルのアウトラインを作る。各節に「その節が主張に対して果たす役割」を1行添える。

完了条件: すべての節に役割が書けている(書けない節は削る)。

### 3. 下書きを書く

アウトラインに沿って本文を書く。この段階から次を守る。

- 具体例・数値・固有名詞を優先し、「近年」「多くの」など出所不明の一般論を書かない
- 箇条書きは列挙が本当に必要な箇所だけ。基本は地の文で書く
- 両論併記で逃げず、書き手としての判断を言い切る

完了条件: 指定分量を満たした全文が揃っている(見出しだけの節が残っていない)。

### 4. 人間らしくリライトする

`~/.claude/skills/rewrite-human/SKILL.md` を読み、その「AI文章の特徴チェックリスト」と「リライトルール」を下書き全文に適用する。文体判定は手順1で決めたものを使う(学術文体ならである調を維持)。

完了条件: チェックリストの全項目を下書きに照らし、該当箇所をすべて書き直した。

### 5. 出力する

リライト後の全文のみを出力する(工程の解説は付けない)。ファイル出力を求められた場合は指定パスに書き出す。
